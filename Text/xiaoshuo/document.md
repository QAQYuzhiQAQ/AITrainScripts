📄 Project Blueprint: "Soul Copyist" - Personal Writing Assistant based on Gemma 31. Project OverviewGoal: Build a local writing assistant that clones the literary style of a specific novel and maintains a deep understanding of its plot/lore.Model Base: Gemma 3 (Local Open-Source)Target: 1. Mimicry of specific prose style.2. Fast, fluid text generation.3. High-fidelity Q&A based on the novel's content.2. Module A: Data Engineering (The "Cook")Cursor Task: Write a Python pipeline to transform raw .txt into training-ready datasets.Task A1: Text Cleaning. Remove artifacts (ads, non-story meta-text) and normalize whitespace.Task A2: SFT Dataset Generation. - Implement a sliding window with overlap (e.g., window=1024 tokens, overlap=256).Format: {"instruction": "...", "input": "...", "output": "..."}.Style extraction: Design prompts that ask the model to continue the story using the specific author's tone.Task A3: Knowledge Extraction. Convert characters, plot points, and world-building rules into a structured FAQ-style JSONL for better logic training.3. Module B: Fine-Tuning Strategy (The "Gym")Cursor Task: Generate training scripts (compatible with unsloth or llama-factory) for Parameter-Efficient Fine-Tuning (PEFT).Method: QLoRA (4-bit quantization) to ensure it runs on local consumer GPUs.Target Layers: Focus on q_proj, v_proj, k_proj, and o_proj.Hyperparameters Reference:Learning Rate: $2 \times 10^{-4}$ (Adjust based on loss).Epochs: 3-5 (Avoid overfitting to specific plot lines).Rank (r): 16 or 32; Alpha: 32 or 64.Validation: Implement a "Style Consistency" check during evaluation (using Perplexity as a metric).4. Module C: RAG System for "Deep Understanding" (The "Library")Cursor Task: Implement a Retrieval-Augmented Generation pipeline to solve the "Memory" issue.Vector DB: Use ChromaDB or FAISS (Local).Embedding Model: Suggest a lightweight local model (e.g., bge-small-zh-v1.5).Logic: When the user asks a factual question about the novel, the system must:Search the Vector DB for relevant context.Inject context into the prompt: [Context: {retrieved_text}] User: {question}.The model (fine-tuned Gemma 3) synthesizes the answer using the retrieved facts + its learned style.5. Module D: Interface & Integration (The "App")Cursor Task: Create a simple Gradio or Streamlit UI for the final user.Mode Toggle: "Creative Writing" (Style-heavy) vs. "Story Consulting" (Fact-heavy).Streaming Output: Ensure real-time tokens generation for a "smooth" feel.Model Loader: Using transformers or AutoGPTQ for efficient loading of the adapter.
